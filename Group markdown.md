# CONCLUSION



 in the first week which is our introduction week we started by learning how to use basic Python tools like `if` statements, loops, and lists. These helped us make simple programs that could do things like check conditions, repeat actions, and store tasks.

Then in week 3 we learned how to write **functions**. Functions let us group code together so we don’t have to repeat it. This made the program easier to read and fix.

After that in week 4, we moved to **classes and objects**. A class is like a blueprint, and an object is something made from that blueprint. We made classes like `Task` and `TaskList` to help organize our tasks. This made the app more structured.

Week 5 is more about our relection of the previous weeks, keeping up with the task and excersice from previous week and then moving to `Testing` and `GUI` where we learn the structuring of testand aiding of program design process, and also learning how to carry out testing after program changes to ensure that code sections which previosly worked are not affected by modification or refactoring.

We also learned about **inheritance and Polymorphism** in week 6, which lets one class take features from another. For example, `RecurringTask` could use everything from `Task` and add its own features. We used **polymorphism** to let different types of tasks behave in their own way, even if they used the same method name.

In week 7, we added **debugging** how to understand and apply techniques to our code, design and implementing persistence following OOP principles and also demonstrating the use of Python property to handle managed attribute.
We learned about **properties**, which let us use methods like simple variables. Then we added **persistence**, which means saving tasks to a file so they don’t disappear when the app closes. We used CSV and Pickle files for this.

Finally We learned about the **SOLID principles and Python exceptions**, which are rules that help us write better code. These rules helped us keep each part of the app focused on one job and made it easier to add new features without breaking anything.
